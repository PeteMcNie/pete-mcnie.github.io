Javascript, DOM and Dev Tools

Technical

Javascripts relationship to HTML and CSS

HTML (Hyper-text-markup-language) is the structure of your webpage. Think of it as the bones and muscles. HTML is a great starting point but not very exciting to look at.

CSS (Cascading-style-sheet) is the styling or 'look' of your webpage. Think of CSS as the design we place on those bones and muscles. With CSS you can have millions of different designs and thats just in the color department. Thinking back to our bones and muscles, CSS gives us the ablility to change the size and shape of those muscles as well as alter all sorts of other parts like the hair and eye color to create a person from those humble beginnings.

But wait, didn't we want to know what Javascripts relationship was with HTML and CSS? Now that we understand how HTML and CSS work together we can look at how Javascript interacts with them both but first we need to understand what the DOM is.

The DOM (Document Object Model)

Firstly, your computer runs a certain type of browser to explore the internet. This could be safari, firefox or the popular google chrome. These browsers create a window for us to use. Inside of this window a document or webpage is created by us, our webpage!

The document is actually a DOM made up of 'NODES' to the computer. What we see as pieces of data such as elements (<body> or <div>) & attributes (class="time") etc the computer sees as NODES in the DOM.

This is where Javascript comes in. Using Javascript we can manipulate the NODES in the DOM, which we see as manipulation of the pieces of data that make up our webpage, the elements and attributes etc.

How does Javascript interact with the DOM to alter our web page?

Using in-built methods that Javascript has, such as:

  • .getElementsByTagName()
  • .getElementsByClassName()
  • .getElementsByID()
  • .getElementsByName()
  • .querySelector()
  • .querySelectorAll()

These methods (and more) can target a NODE, or multiple NODES and we can then alter the NODE in many different ways. In fact we can alter NODES in so many ways we do not have time to mention them all here but once Javascript is targeting a NODE in the DOM we can add interactivity to our page such as adding events on mouse clicks, pop ups and auto-filling of forms etc.

Summary

We can see that our HTML and CSS can be enhanced and altered by Javascript. This gives our boring page interactivity. Our initial person can be turned into a chef, banker, police officer or web developer. By making the web page have special skills and being creative our web page can be turned into a unique experience for guests.

DevTools

What are DevTools and how do they relate to the DOM?

Chrome Developer Tools or Dev Tools as they are known, are found by right clicking on a web page and selecting inspect from the list provided. The developer tools will open up with the Elements heading selected at the top of the page. This is a list of elements that make up the DOM tree. It looks very similar to our HTML however there are some differences.

Using DevTools we can search for and select different NODES that make up our web page. From here we are able to edit the DOM directly and see how those changes affect the web page in real time. Normally any changes we make are wiped the second we reload the web page from the oringal source. However, we are able to give DevTools access to our web page files (if we have them) which makes any changes we make permanent. This is a handy tool for developers trying to alter their page and see results we real time.

Control Flow

Control flow is simply the order in which the computer will execute the code we have written. Code normally runs from the first line to the last line, unless it runs across any type of code that alters the flow of the code such as conditionals and loops. These are types of control flow.
